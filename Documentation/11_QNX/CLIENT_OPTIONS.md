# Варианты клиента для Remote Tunnel

## Текущий вариант: Shell скрипт

**Файлы:**
- `/data/local/tmp/remote_tunnel_client.sh`
- `/data/local/tmp/remote_tunnel_client_v2.sh`
- `/data/local/tmp/remote_tunnel_client_v3.sh`

**Характеристики:**
- ✅ Простой, не требует компиляции
- ✅ Легко редактировать и тестировать
- ❌ Требует ADB для запуска
- ❌ Не запускается автоматически при загрузке
- ❌ `busybox nc` не поддерживает постоянное соединение
- ❌ Может быть убит системой при нехватке памяти

**Запуск:**
```bash
adb shell "nohup /data/local/tmp/remote_tunnel_client.sh &"
```

---

## Рекомендуемый вариант: Android Service (Java)

**Файлы:**
- `ReverseTunnelClient.java` - основной сервис
- `BootReceiver.java` - автозапуск при загрузке
- `AndroidManifest.xml` - конфигурация

**Характеристики:**
- ✅ Автозапуск при загрузке через BroadcastReceiver
- ✅ Постоянное соединение через Java Socket
- ✅ Работает как системный сервис в фоне
- ✅ Не требует ADB для работы
- ✅ Устойчив к перезапускам
- ✅ Поддержка screen capture и touch input
- ❌ Требует компиляции в APK
- ❌ Требует установки на устройство

**Компиляция и установка:**
```bash
# Создать Android проект
# Скомпилировать APK
# Установить через ADB
adb install remote_tunnel_client.apk
```

**Преимущества:**
1. **Автозапуск:** Сервис запускается автоматически при загрузке через `BootReceiver`
2. **Постоянное соединение:** Java Socket поддерживает постоянное соединение с keep-alive
3. **Устойчивость:** Система не убивает сервисы так легко, как shell скрипты
4. **Функциональность:** Встроенная поддержка screen capture и touch input

---

## Рекомендация

**Использовать Android Service (Java)** для production использования, так как:
- Автоматически запускается при загрузке
- Поддерживает постоянное соединение
- Более надежен и устойчив

**Shell скрипт** можно использовать для:
- Быстрого тестирования
- Отладки
- Временных решений
