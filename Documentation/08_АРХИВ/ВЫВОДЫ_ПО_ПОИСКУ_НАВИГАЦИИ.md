# Выводы по поиску приложения навигации

## Результаты поиска

### Найдено:
1. **SVMapService** (`com.desaysv.mapservice`)
   - Системный сервис навигации
   - НЕ содержит Activity (только Service)
   - Поддерживает AutoNavi и Thinkware через Broadcast

2. **SVMeterInteraction** (`com.desaysv.meterinteraction`)
   - Взаимодействие с приборной панелью
   - НЕ является приложением навигации

### Не найдено:
- Само приложение навигации (TurboDog/AutoNavi)
- Package name приложения "НАВИГАЦИЯ" из лаунчера
- Упоминания `com.autonavi`, `com.amap`, `com.turbodog` в системных приложениях

---

## Выводы

### 1. Приложение навигации НЕ находится в extracted системных приложениях
- Все системные приложения из образа уже извлечены
- Приложение навигации, вероятно, устанавливается отдельно или находится в другом месте

### 2. SVMapService - это мост между навигатором и системой
- SVMapService получает данные от навигатора (через Broadcast)
- SVMapService отправляет данные в систему (через Broadcast или другие механизмы)
- Само приложение навигации - это отдельное приложение (TurboDog/AutoNavi)

### 3. Возможные варианты расположения приложения:
1. **Установлено как пользовательское приложение:**
   - Не находится в системных приложениях
   - Может быть установлено через обновление или отдельно
   - Package name: `com.autonavi.minimap` или `com.autonavi`

2. **Находится в обновлении прошивки:**
   - Может быть в `soc_update.zip`
   - Нужно проверить содержимое обновления

3. **Встроено в систему другим способом:**
   - Может быть частью системного образа
   - Может быть установлено в `/system/priv-app/` или `/system/app/`

## Следующие шаги

### Немедленно:
1. ⏳ Проверить содержимое `soc_update.zip` на наличие APK навигации
2. ⏳ Поиск по известным package names AutoNavi в других местах
3. ⏳ Проверить, может ли приложение быть установлено отдельно

### Альтернативный подход:
Если приложение навигации не найдено, можно:
1. **Использовать известный формат коммуникации:**
   - Известен формат Broadcast Intent: `turbodog.navigation.system.message`
   - Можно отправлять данные в этом формате без знания самого приложения

2. **Изучить, как SVMapService получает данные:**
   - Какие Broadcast Intents слушает SVMapService?
   - Какой формат данных ожидает?

3. **Создать Bridge Service:**
   - Bridge Service будет получать данные от Яндекс Навигатора
   - Преобразовывать в формат TurboDog
   - Отправлять через известный формат Broadcast Intent
