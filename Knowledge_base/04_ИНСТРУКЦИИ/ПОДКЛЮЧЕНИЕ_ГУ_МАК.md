# Подключение ГУ к Android Studio на Mac (из Windows VM)

## Ваша конфигурация:
- **Mac (хост)**: Android Studio установлен здесь
- **Windows VM (Parallels)**: Cursor работает здесь
- **ГУ**: Нужно подключить к Android Studio на Mac

## Рекомендуемый способ: Подключение через Wi-Fi

Это самый удобный способ, так как не зависит от того, к чему подключен USB кабель.

### Шаг 1: Первоначальная настройка (один раз)

1. **Подключите ГУ к Mac через USB** (можно через Parallels, если настроен USB passthrough)
2. **На Mac откройте Terminal** и выполните:
   ```bash
   adb tcpip 5555
   ```
   Это переведет ADB в режим TCP/IP на порту 5555

### Шаг 2: Узнайте IP адрес ГУ

**На ГУ:**
1. Откройте **Настройки** → **Wi-Fi**
2. Нажмите на текущую подключенную сеть
3. Найдите **IP адрес** (например, `192.168.1.100`)

### Шаг 3: Подключитесь по Wi-Fi

**На Mac в Terminal:**
```bash
adb connect <IP_АДРЕС>:5555
```
Например: `adb connect 192.168.1.100:5555`

### Шаг 4: Проверка в Android Studio

1. Откройте **Android Studio на Mac**
2. Перейдите: **View** → **Tool Windows** → **Device Manager**
3. Ваше устройство должно появиться в списке с пометкой **(Wi-Fi)**

## Альтернатива: USB подключение напрямую к Mac

Если у вас есть USB кабель и вы можете подключить ГУ напрямую к Mac:

1. **Подключите ГУ к Mac через USB**
2. **Включите USB отладку на ГУ** (см. инструкцию ниже)
3. **В Android Studio на Mac** устройство должно появиться автоматически

## Включение USB отладки на ГУ

1. **Настройки** → **О системе** → Нажмите **7 раз** на "Номер сборки"
2. **Настройки** → **Для разработчиков** → Включите **"USB отладка"**
3. При подключении разрешите отладку по USB

## Настройка USB в Parallels (если нужно)

Если вы хотите подключить ГУ к Windows VM:

1. В Parallels: **Устройства** → **USB** → Выберите ваше ГУ
2. ГУ будет доступно в Windows VM
3. В Windows VM можно использовать ADB для отладки

## Проверка подключения

**На Mac в Terminal:**
```bash
adb devices
```

Должно показать:
```
List of devices attached
192.168.1.100:5555    device
```
или
```
ABC123XYZ    device
```

## Устранение проблем

### Устройство не видно в Android Studio на Mac

1. Проверьте, что ADB запущен:
   ```bash
   adb kill-server
   adb start-server
   adb devices
   ```

2. Проверьте, что ГУ и Mac в одной Wi-Fi сети

3. Проверьте, что порт 5555 не заблокирован файрволом

### ADB не найден на Mac

Установите Android SDK Platform Tools:
```bash
brew install android-platform-tools
```

Или скачайте с официального сайта Android и добавьте в PATH.

## После успешного подключения

1. В Android Studio на Mac выберите ваше устройство
2. Запустите приложение: **Run** → **Run 'app'** (⌘R)
3. Приложение установится и запустится на ГУ
